# Norma AI - WhatsApp Orchestrator (Render)

This directory contains the standalone WhatsApp chatbot orchestrator for **Norma AI**. It is designed to run in a cloud environment (like Render) to handle real-time communication between Twilio and the Gemini AI engine.

## 🚀 Overview

The WhatsApp Orchestrator serves as the bridge between patients and the clinic's clinical services. It uses FastAPI for high-performance webhook handling and Gemini 1.5 Flash for natural language understanding and tool execution.

## 🛠 Tech Stack

- **Framework:** FastAPI
- **Database:** MongoDB (via Motor/Motor-AsyncIO)
- **AI Engine:** Google Gemini 1.5 Flash
- **Messaging:** Twilio WhatsApp API
- **Runtime:** Python 3.10+

## 📁 Architecture

- `app/api/webhook.py`: Entry point for Twilio webhooks. Handles incoming traffic and background task dispatching.
- `app/services/ai_service.py`: Core logic for intent parsing, patient recognition, and clinical tool execution (booking, registration, etc.).
- `app/services/whatsapp_service.py`: Integration layer with Twilio for sending outbound messages.
- `app/db/mongodb.py`: Database connection management and indexing logic.

## 🔧 Setup & Environment Variables

Create a `.env` file in this directory with the following keys:

```env
MONGODB_URI=your_mongodb_connection_string
MONGODB_DB_NAME=norma_ai
GEMINI_API_KEY=your_google_ai_studio_key
TWILIO_ACCOUNT_SID=your_twilio_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_PHONE_NUMBER=your_twilio_whatsapp_number
```

## 📊 Logging & Monitoring

The system implements structured logging for every interaction, visible in the Render terminal:

### Incoming (Twilio -> Server)
Logs include the sender's phone, the receiver number, the message body, and the unique Twilio SID.

### Outgoing (Server -> Twilio)
Logs include the destination number, source number, and the exact response text generated by the AI.

### Error Tracking
Full tracebacks are logged for any exceptions occurring during AI processing or message dispatch.

## 🏥 Clinical Capabilities

The chatbot currently supports:
- **Patient Registration:** Auto-recognizes existing users by phone number.
- **Appointment Booking:** Real-time doctor selection and slot checking.
- **Appointment Management:** Viewing upcoming visits and cancellation.
- **Natural Language:** Powered by Gemini 1.5 Flash for human-like medical assistance.
